Object Oriented Programming in C and C++

Introduction to Object Oriented Programming (OOP)

  • OOP involves creating a simulation of the problem environment using objects representing different parts of the system.
  • Objects have states (data/properties) and behaviors (functions) that interact to efficiently solve problems.
  • Classes in OOP are similar to structures but allow integration of functions into the data structure to provide behaviors.
  • Objects are instances of classes with properties and behaviors specific to that instance of the class.
  • Access to class members can be specified as public, private, or protected, controlling visibility and modification abilities.
  • Encapsulation hides internal implementation details from users, allowing access only through public interfaces.

Example:


class Circle {
private:
    float radius;
public:
    void print(); // Example behavior
};

Creating Classes and Objects

  • Syntax: class ClassName { };
  • Classes encapsulate data (variables) and behaviors (functions).
  • Use public interfaces to allow access to class members.
  • Objects of classes are created by instantiating them where memory gets allocated for their properties and functions.

Example:


class Fraction {
private:
    int numerator;
    int denominator;
public:
    void print(); // Function to print fraction
};

Fraction f1, f2; // Creating objects of the class
f1.print(); // Accessing behavior of the object f1

Constructors and Destructors

  • Constructors initialize objects when created, matching the class name.
  • Default constructors have no arguments and can have default parameter values.
  • Destructors clean up resources when objects go out of scope.
  • Constructors and destructors don't have return types and can be overloaded with different signatures.

Example:


class Fraction {
private:
    int numerator;
    int denominator;
public:
    void print(); // Function to print fraction

    //constructors
    
    Fraction() {
        numerator = 1;
        denominator = 1;
    }
    Fraction(int n, int d) {
        numerator = n;
        denominator = d;
    }

    // return the fraction 

    double getFraction()
    {
        return numerator / denominator;
    }
};

Fraction f1, f2; // Creating objects of the class
f1.print(); // Accessing behavior of the object f1

Access Modifiers and Encapsulation

  • scope resolution operator (::) can be used so that functions of a class can be accessed from outside
    • EX: 
void clockType::incrementSeconds(){
    seconds++;

    {
        seconds = 0;
        incrementMinutes();
    }
}
  • Access modifiers control the visibility and modification permissions of class members.
  • Structs are public by default, while classes are private.
  • Encapsulation hides internal details, allowing interactions only through specified public interfaces for better software design.

General guidelines for public interface:

  • Public interface is a set of functions that can be called by other classes.
    • Accessor functions and mutator functions
      • Mutator functions can be used to change the state of the object.
      • Accessor functions cannot
  • Private interface is a set of functions that cannot be called by other classes.
  • Protected interface is a set of functions that can be called by other classes, but cannot be called by other classes.
  • NOTE: the order in which you declare public, private, or protected interfaces does not matter but if it is not declared, private is implied by default.

Constructors

  • Constructor is a function that is called when an object is created.
  • Two types:
    • Default constructor - contains no parameters
    • Copy constructor - contains parameters
  • properties of constructors:
    • it is a function with no type
    • a class can have multiple constructors but they all have the same name
    • all constructors must have different parameter lists
    • constructors execute automatically when a class object enters scope
    • which constructors executes depends on the passed values

Key Takeaways:

  • OOP involves modeling real-world scenarios using objects with data and behavior.
  • Most of C++ built in operations aren't supported on classes
    • You may, however, access members with the dot operator for access and assign values with '='
  • Classes define the blueprint for objects, and objects are instances of classes.
  • Access modifiers control the visibility and access levels of class members.
    • NOTE: if a class type object accesses another object of the same type, private members are accessible.
  • Constructors initialize objects, and destructors clean up resources when objects are no longer needed.
    • Destructors are calles automatically when an object goes out of scope. Only one need be made.
  • Encapsulation hides internal implementations and enforces interactions through public interfaces.
  • Custom header files must be referenced with the .h extension
    • However, you can also simply create objects and reference them within a single file.
