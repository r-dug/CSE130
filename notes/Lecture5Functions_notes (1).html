Lecture 5: Functions in CSE 130

Overview of Functions

  • Functions allow for code to be broken into smaller, reusable units.
  • They help in managing complexity by dividing a project into manageable tasks.
  • Increases maintainability, reduces debugging, and facilitates code reuse.
  • Functions serve as black boxes with clear inputs and outputs.

Advantages of Using Functions

  • Reusability: Functions can be shared and reused across different parts of a program or by different programs.
  • Information Hiding: Functions hide the details of their internal implementation which prevents data manipulation errors.
  • THIS WILL BE ON THE EXAM: Top-down design approach and importance of a function doing just one thing well.

Function Definition

  • A function is defined by:
    • A name
    • Parameter list
    • Return type
  • A function prototype informs the compiler of the function's existence prior to its actual definition.
  • Prototype Format: return_type function_name(parameter_list);
  • Examples: printf, scanf, mathematical operations, etc.

Function Prototypes and Calls

  • Prototypes specify the types of the parameters and the function's return type.
  • A function call is made by passing arguments to a function which then returns a value.
  • Function arguments can be constants, variables, or expressions.

Passing Parameters to Functions

  • Pass by Value: Copies the actual value to the function, no change to the original variable.
  • Pass by Reference: Passes address of variable so changes within the function affect the original variable.
  • Scope of Variables: Visibility and lifetime of a variable. Local inside functions/blocks, global outside.

Variable Scope and Storage Classes

  • Local Variables: Declared within a function, visible only within the function's scope.
  • Global Variables: Declared outside functions, visible throughout the entire program.
    Use of global variables is discouraged due to security and safety reasons.
  • Static Variables: Retain their values even after the function in which they are declared is finished executing.
  • Storage Classes:
    • Auto: The default storage class for local variables.
    • Static: Variables retain their value between function calls.
    • Extern: Used to declare a global variable or function in another file.
    • Register: Hint to the compiler to store variable in register memory for quick access.

Final Notes

  • The importance of using functions properly and passing arguments by value or by reference is emphasized.
  • Global variables should be avoided in favor of local variables for security purposes.
